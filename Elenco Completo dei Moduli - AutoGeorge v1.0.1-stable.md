## 3. COME I MODULI LAVORANO INSIEME

### Flusso tipico di creazione articolo:
1. **Dashboard** ‚Üí Utente avvia generazione tramite `/admin/generate` page
2. **API Endpoint** ‚Üí `/api/admin/generate-article` riceve la request
3. **Composition Root** ‚Üí Container DI risolve le dipendenze (ContentAdminFacade)
4. **Content Module** ‚Üí `GenerateArticle` use case elabora il prompt
5. **AI Service** ‚Üí Chiamata a Perplexity API per generazione contenuto
6. **Article Repository** ‚Üí Salvataggio nel database PostgreSQL
7. **Response** ‚Üí Articolo generato ritorna al frontend con metadati

### Flusso gestione sources:
1. **Sources Admin** ‚Üí Configurazione fonte tramite `/admin/sources`
2. **API Endpoint** ‚Üí `/api/admin/sources` gestisce CRUD operations
3. **Sources Module** ‚Üí Use cases (CreateSource, GetSources, FetchFromSource)
4. **External Adapters** ‚Üí Connessione RSS feeds, Telegram channels
5. **Database** ‚Üí Persistenza configurazioni e metadati fetch
6. **Monitoring** ‚Üí Health checks e statistiche sources

### Flusso automatico polling:
1. **Cron Job** ‚Üí `/api/cron/poll-feeds` attivato periodicamente
2. **Feed Polling Service** ‚Üí Recupera contenuti da tutte le fonti attive
3. **FeedItem Storage** ‚Üí Salva nuovi elementi nel database
4. **Article Generation** ‚Üí Trigger automatico per articoli da feed items
5. **Debug Monitoring** ‚Üí `/api/debug/rss-logs` per troubleshooting
